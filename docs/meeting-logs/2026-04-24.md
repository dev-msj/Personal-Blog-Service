1. MCPSI 파이프라인으로 Phase 재수립 진행 (신규 수립)
   - 맥락: 기능 미완 + 구조적 완성도 부족 + 이슈 파편화 상태에서 임시방편으로 이슈를 소화하는 대신, 근거 있는 Phase 분리를 수립해 학습 가치와 추적성을 확보할 필요가 있음
   - 근거: 학습 프로젝트의 핵심 산출물은 "결정의 근거 문서화"이며, MCPSI는 Problem 단계에서 Phase 분리 근거를 정식 수립하는 구조를 제공. 기존 이슈를 감으로 재배치하는 것보다 정합성이 높음
   - 기각 대안: (1) 기존 이슈만 순차 소화 — 학습 가치 낮고 Phase 관점 누락. (2) 즉흥적 Phase 분할 — 근거 미기록으로 사후 추적 불가
   - MCPSI 영향: C/P/S 전체. context에 현 코드/이슈 상태 분석 포함, problem에 Phase 분리 근거 수립, solution에 Phase별 설계

2. 기존 GitHub 이슈는 Phase 재수립 결과에 따라 재구성 대상으로 취급

   - 기각 대안: Phase 수립과 무관하게 선제적 이슈 정리 — 재작업 가능성 높음
   - MCPSI 영향: 직접 영향은 Implementation 단계. Context 단계에서 현재 이슈 상태를 입력으로 포함
