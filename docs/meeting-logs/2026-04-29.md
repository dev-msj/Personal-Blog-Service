## plan-manager 연계

본 미팅 로그는 다음 plan-manager 트리거에 해당하지 않음:
- Phase 전환 결정 — 해당 없음 (Phase 0 진행 중)
- 정책 변경 (small-first ↔ core-first) — 해당 없음
- 작업자 수 변경 — 해당 없음
- plan 재정렬 사유 — problem.md / solution/overview.md 재작성이 plan 트랙 라인의 마일스톤이나 의존 관계를 변경하지 않음
- 이관 이슈 발생 패턴 변화 — 해당 없음

따라서 본 미팅 로그 작성 후 plan-manager 호출 불필요. mcpsi 재실행은 problem 단계부터 (context는 입력 변경 없음, problem 정책 미준수 해소가 본 미팅 결정의 핵심).

## mcpsi 파이프라인 재실행 범위

- context: 재실행 불필요 (도메인/제약/배경 변경 없음)
- problem: **재실행 필요** (Phase 근거 섹션 신정책 적용)
- solution: 일부 재실행 — overview.md는 problem.md 변경 결과를 반영하여 갱신 필요. Extension 문서는 cross-reference 정합 검사 후 충돌 시에만 재실행
- implementation: 재실행 불필요 (issue-plan.md 운영 정의가 문서로 역승격되는 흐름이지, plan 자체 변경 아님)

권장 순서: /mcpsi-problem → /mcpsi-problem-verify → /mcpsi-solution-core (overview.md 갱신만, Extension은 정합 검사 후 결정) → 필요 시 Extension 재실행
