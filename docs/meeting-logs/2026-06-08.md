## 원본 기록 (발산)

### 주제 1: docs/meta 거버넌스 3종의 유지 비용 재평가

- 2026-05-11 미팅 로그(Phase 0→1 전환 + C/P/S/I 재유입) 결정에 따라 2026-05-13 commit 11213d5로 docs/meta 거버넌스 3종 도입: Traceability Matrix(docs/meta/traceability.md) + ADR 라이프사이클(adr/) + Change Impact Log(change-impact-log.md)
- Traceability Matrix는 9컬럼(Stakeholder Req → FR/NFR/QAS → ADR → 모듈 → API/Event → 다이어그램 → 테스트 → 운영 메트릭 → 상태)으로 단계 횡단 8단 추적을 운용
- 단계 종료마다 전 컬럼 동기화를 요구하나, RT-0001~0005 다수 셀이 "Solution/Implementation/Observability 단계 미진행 사유 명시"로 채워진 상태. 정보 밀도 대비 유지 비용 과다
- 1인 학습 프로젝트에서 9컬럼 매트릭스의 단계별 전 컬럼 재동기화는 실질 추적 가치를 초과하는 부담으로 판단

### 주제 2: 추적 가치의 실질 커버 주체

- 매트릭스가 제공하려던 핵심 가치는 (a) 결정의 근거 추적, (b) 변경의 영향 범위 추적
- (a)는 ADR 라이프사이클(adr/_index.md + ADR-0001~0015 MADR 4.0 본문)이 이미 전담
- (b)는 Change Impact Log(append-only 누적 로그)가 이미 전담
- 즉 매트릭스의 추적 가치는 잔존 2종으로 흡수 가능하며, 매트릭스는 두 산출물을 한 표로 묶어 보여주는 중복 뷰 성격이 큼

### 주제 3: 폐지 차원 — 프로젝트 운영 vs 방법론

- 폐지는 본 프로젝트가 traceability.md 파일을 운영하는 것에 한정
- mcpsi-traceability 스킬 자체는 유지된다. 스킬은 잔존하는 ADR 라이프사이클 + 변경 임팩트 매트릭스의 단일 운영자로 계속 기능하며, traceability-format.md 레퍼런스도 살아있어 향후 재도입/타 프로젝트 적용 가능
- 따라서 방법론 차원의 Traceability Matrix 개념은 폐지되지 않음. 본 프로젝트의 운영만 중단

### 주제 4: 폐지 후 잔존 문서의 참조 정합성 검토

## 결정 사항 (수렴)

1. 요구 추적 매트릭스(docs/meta/traceability.md) 운영 폐지
   - 맥락: 2026-05-13 도입한 docs/meta 거버넌스 3종 중 Traceability Matrix의 유지 비용이 1인 학습 프로젝트에서 추적 가치를 초과한다고 재평가
   - 근거: 9컬럼 매트릭스의 단계별 전 컬럼 동기화 비용이 과다하고, 다수 셀이 "단계 미진행 사유"로 채워져 정보 밀도가 낮음. 매트릭스가 제공하던 결정 근거 추적은 ADR 라이프사이클이, 변경 영향 추적은 Change Impact Log가 이미 전담하므로 매트릭스는 중복 뷰. 폐지해도 추적 가치 손실 없음
   - 기각 대안: (1) 경량화 유지(컬럼 축소) — 컬럼을 줄여도 단계별 동기화 의무가 잔존하고 ADR/Change Impact Log와의 중복이 해소되지 않음. (2) 동결(파일은 유지하되 갱신 중단) — 갱신 멈춘 stale 문서가 authoritative처럼 잔존하여 오히려 정합성 혼란을 유발. 삭제가 더 명확
   - MCPSI 영향: 메타 거버넌스 영역[국소] — C/P/S/I 단계 산출물에 영향 없음. Traceability Matrix는 산출물을 추적하는 뷰일 뿐 어느 단계의 입력도 아니므로 단계간 재통과 트리거 미발화

2. ADR 라이프사이클 + Change Impact Log 잔존 확정
   - 맥락: 거버넌스 3종 중 매트릭스만 폐지하고 나머지 2종의 존속을 명시
   - 근거: 결정 근거 기록(ADR)은 본 학습 프로젝트의 핵심 산출물이고, 변경 파급 추적(Change Impact Log)은 append-only로 저비용. 둘 다 매트릭스 없이 독립적으로 가치를 가짐
   - 기각 대안: 거버넌스 3종 전체 폐지 — ADR/Change Impact Log는 비용 대비 가치가 명확하여 함께 폐지할 이유 없음. 결정 근거 문서화라는 프로젝트 목표에 직접 기여
   - MCPSI 영향: 없음 (현행 유지 확인). adr/_index.md, change-impact-log.md, mcpsi-traceability 스킬 description(docs/meta/* 단일 운영자) 모두 정합

3. 폐지 범위는 본 프로젝트 운영 차원 — mcpsi-traceability 스킬은 유지
   - 맥락: 폐지가 방법론(스킬) 차원인지 프로젝트 운영 차원인지 명확화
   - 근거: mcpsi-traceability 스킬은 ADR 라이프사이클 + 변경 임팩트 매트릭스 운영자로 계속 기능하고 traceability-format.md 레퍼런스도 잔존. 본 프로젝트가 traceability.md 파일을 더 이상 운영하지 않을 뿐, 방법론에서 매트릭스 개념이 제거된 것은 아님
   - 기각 대안: 없음 (범위 명확화 성격 결정)
   - MCPSI 영향: 없음 (스킬 변경 아님). 향후 동일 매트릭스 재도입이 필요하면 신규 미팅 로그로 결정

4. design-doc-as-code-mcpsi 블로그에 회고 각주 보강
   - 맥락: 블로그가 Traceability Matrix를 방법론 보강 6영역 중 하나로 서술. 폐지 후 독자 오독 가능성 검토
   - 근거: 블로그 본문(방법론 설계 서사)은 여전히 사실이므로 재작성은 과함. 다만 "본 프로젝트에서 운영 중"으로 읽힐 여지가 있어, 회고/한계 맥락에 "이후 거버넌스 정책 변경으로 본 프로젝트는 요구 추적 매트릭스 운영을 중단하고 ADR + 변경 임팩트 로그만 유지한다"는 각주 1줄 추가가 정합성 비용 최소 방안
   - 기각 대안: (1) 본문 재작성 — 방법론 서사 자체는 유효하므로 과잉. Documentation Debt 증가. (2) 무수정 — 오독 여지 잔존. 각주 1줄로 해소 가능한 비용을 방치
   - MCPSI 영향: 산출물(블로그) 국소 — docs/tech-notes/design-doc-as-code-mcpsi/README.md 회고 섹션에 각주 추가. CLAUDE.md 블로그 규칙("결정 번복 시 MCPSI부터 수정 후 블로그 재작성")의 경량 적용
