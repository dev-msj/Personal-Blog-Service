## mcpsi 파이프라인 재실행 범위

- meeting-log: 본 로그가 입력
- meeting-log-verify: 본 로그 검증
- context: 재실행 (스킬 정책 정합 + Phase 1 입력 정합성 검토)
- context-verify: 재실행 결과 검증
- problem: 재실행 (스킬 정책 정합 + Phase 1 범위 재검토)
- problem-verify: 재실행 결과 검증
- solution: 재실행 (스킬 정책 정합 + Phase 1 솔루션 재검토 + 신규 Extension 적용 조건 판정)
- solution-verify: 재실행 결과 검증
- implementation: Phase 1 대상 신규 분해
- traceability: 단계 종료마다 갱신
